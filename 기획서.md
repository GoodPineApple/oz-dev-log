개발자의 성장을 돕는 **성장일지(TIL: Today I Learned)** 서비스의 핵심은 '기록의 진입장벽을 낮추는 것'과 '꾸준함을 시각화하는 것'입니다. 가장 핵심적인 기능만 담은 MVP(Minimum Viable Product) 기획문서를 정리해 드립니다.

---

## 1. 프로젝트 개요

- **서비스명:** DevLog (가칭)
- **목표:** 개발자가 매일 배운 내용을 5분 안에 기록하고, 자신의 성장 궤적을 확인하는 서비스
- **핵심 가치:** 간결함(Simplicity), 기록의 습관화(Habit), 회고(Reflection)

## 2. 핵심 기능 (MVP Scope)

| 구분     | 기능                | 상세 설명                                                |
| :------- | :------------------ | :------------------------------------------------------- |
| **인증** | 소셜 로그인         | 구글 계정을 활용한 간편 로그인 (Firebase Auth 활용 권장) |
| **기록** | 오늘 배운 내용 작성 | 마크다운(Markdown) 에디터를 지원하는 간단한 글쓰기 환경  |
| **조회** | 나의 일지 목록      | 날짜별 역순으로 정렬된 일지 리스트 (무한 스크롤)         |
| **상세** | 일지 상세 보기      | 작성한 내용을 깨끗한 UI로 확인                           |
| **관리** | 수정 및 삭제        | 오타 수정이나 불필요한 기록 삭제                         |

---

## 3. 기술 스택 제안

빠른 배포와 확정성을 고려하여 다음 스택을 제안합니다.

- **Frontend:** `Next.js` (App Router 기반, SEO 및 빠른 초기 로딩)
- **Styling:** `Tailwind CSS` (빠른 UI 구현)
- **Backend/DB:** `Firebase` (Firestore: 문서 기반 NoSQL, 빠른 데이터 연동)
- **Deployment:** `Vercel` (Next.js 최적화 배포 환경)

---

## 4. 데이터베이스 설계 (Firestore 구조)

### **`users` Collection**

- `uid`: string (Primary Key)
- `email`: string
- `displayName`: string
- `photoURL`: string

### **`logs` Collection**

- `id`: string (Auto-generated)
- `userId`: string (Reference to users.uid)
- `title`: string (오늘의 한 줄 요약)
- `content`: string (마크다운 형식의 상세 내용)
- `tags`: array (예: #React, #Nextjs)
- `createdAt`: timestamp

---

## 5. 사용자 시나리오 (User Flow)

1.  **접속 및 로그인:** 사용자가 서비스 접속 후 구글 버튼 클릭으로 로그인 완료.
2.  **홈 화면:** 내가 작성한 최신 일지 목록이 타임라인 형태로 표시됨.
3.  **작성:** '오늘의 기록하기' 버튼을 눌러 제목과 내용을 입력하고 저장.
4.  **확인:** 저장 즉시 리스트 최상단에 업데이트되며, 클릭 시 상세 내용을 확인.

---

## 6. UI/UX 가이드라인

- **미니멀리즘:** 불필요한 사이드바나 광고를 배제하고 글쓰기 본연에 집중하는 화이트 톤의 클린 UI.
- **반응형 레이아웃:** 모바일에서도 배운 즉시 기록할 수 있도록 모바일 우선 디자인(Mobile-First) 적용.
- **에디터 편의성:** 작성 중 자동 저장(Auto-save) 기능은 제외하더라도, 마크다운 미리보기 기능은 포함하여 작성 경험 향상.

---

## 7. 향후 확장 계획 (Post-MVP)

- **잔디(Contribution Graph):** 깃허브처럼 일자별 기록 현황을 시각화.
- **태그 검색:** 특정 기술 스택별로 모아보기.
- **공유 기능:** 특정 일지를 외부(SNS, 블로그)에 공유할 수 있는 링크 생성.
- **AI 요약:** 작성한 긴 내용을 한 문장으로 요약해 주는 AI 기능 도입.

가장 먼저 어떤 기능부터 개발을 시작해보고 싶으신가요?

사용자가 단순 텍스트를 넘어 **풍부한 기록(이미지, 코드, 파일)**을 남기고, 이를 **보상(크레딧)**으로 연결하는 구조는 서비스의 유지력(Retention)을 높이는 훌륭한 전략입니다.

추가된 요구사항을 반영한 **업데이트된 MVP 기획문서**입니다.

---

## 1. 확장된 핵심 기능 (Scope Expansion)

### **A. 멀티미디어 첨부 시스템**

- **파일 업로드:** 드래그 앤 드롭 또는 파일 선택을 통해 기록 당 최대 3~5개의 파일 첨부 가능.
- **지원 형식:**
  - **이미지:** `.png`, `.jpg`, `.gif` (기록의 시각화)
  - **코드/압축:** `.js`, `.py`, `.zip` 등 (프로젝트 소스 보존)
- **스토리지:** 파일 용량 제한(예: 개당 10MB)을 두어 초기 비용 관리.

### **B. 크레딧 시스템 (Gamification)**

- **기본 로직:** 활동 지수에 따라 크레딧을 차등 지급하여 지속적인 기록 유도.
- **크레딧 획득 규칙 (예시):**

| 활동 항목          | 획득 크레딧 | 비고                              |
| :----------------- | :---------- | :-------------------------------- |
| **일지 신규 작성** | 100 CP      | 1일 1회 한정 (도배 방지)          |
| **파일 첨부 시**   | +20 CP      | 이미지나 파일 포함 시 추가 보너스 |
| **연속 기록 달성** | +50 CP      | 3일/7일 등 연속 작성 시 보너스    |
| **글자 수 충족**   | +10 CP      | 300자 이상 작성 시 성실도 점수    |

---

## 2. 기술 스택 및 인프라 변경

첨부 파일 처리를 위해 기존 스택에 **Storage** 기능을 추가합니다.

- **Storage:** `Firebase Storage` (파일 업로드 및 보안 규칙 적용)
- **Database:** `Firestore` (사용자별 크레딧 필드 및 파일 경로 저장)
- **State Management:** `React Query` 또는 `SWR` (파일 업로드 상태 및 크레딧 실시간 반영)

---

## 3. 업데이트된 데이터베이스 설계 (Firestore)

### **`users` Collection**

- `uid`: string
- **`totalCredits`**: number (누적 크레딧)
- **`currentLevel`**: number (크레딧 기반 레벨)
- `lastPostedAt`: timestamp (연속 작성 체크용)

### **`logs` Collection**

- `id`: string
- `userId`: string
- `title`: string
- `content`: string
- **`attachments`**: array (Storage의 다운로드 URL 리스트)
  - `{ type: 'image', url: '...', name: 'shot.png' }`
  - `{ type: 'file', url: '...', name: 'project.zip' }`
- **`earnedCredits`**: number (해당 로그로 획득한 점수)
- `createdAt`: timestamp

---

## 4. 사용자 경험 (UX) 강화 포인트

1.  **업로드 프리뷰:** 이미지는 썸네일로 보여주고, 코드 파일은 파일 아이콘과 파일명을 표시하여 직관성 제공.
2.  **크레딧 팝업:** 기록 완료 후 "축하합니다! 120 크레딧을 획득했습니다."와 같은 토스트 메시지 노출.
3.  **마이 페이지(대시보드):** 현재까지 모은 총 크레딧과 보유한 파일 리스트를 한눈에 확인.

---

## 5. 크레딧 활용 방안 (Post-MVP)

모은 크레딧을 단순히 숫자로 두지 않고 활용할 수 있는 장치를 마련합니다.

- **AI 요약 권한:** 1,000 크레딧 소모 시 AI가 한 달간의 기록을 리포트로 생성.
- **테마 잠금 해제:** 특정 크레딧 도달 시 개발자용 다크 모드 테마나 특수 폰트 제공.
- **PDF 내보내기:** 기록한 내용을 포트폴리오용 PDF로 변환하는 기능 잠금 해제.

---

## 6. 개발 우선순위 (Sprint 1)

1.  **[1순위]** Firebase Storage 연동 및 기본 파일 업로드/조회 기능 구현.
2.  **[2순위]** 로그 저장 로직에 크레딧 계산 로직(Cloud Functions 혹은 Client Side) 추가.
3.  **[3순위]** 메인 화면에 내 현재 크레딧 상태 표시 UI 구현.
